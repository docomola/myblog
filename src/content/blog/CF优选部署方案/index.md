# CF优选部署方案

​`edgetunnel` 的本质是利用 Cloudflare Workers/Pages 的无服务器（Serverless）架构，将 VLESS / Trojan 等协议的流量通过 WebSocket 转发出去。

因为这个方案非常火，开源社区里诞生了很多同类、甚至比 `edgetunnel`​ 功能更强大的优秀开源项目。它们各有侧重点，​**没有绝对的“哪个最好”，只有“哪个最适合你的需求”** 。

为你整理了目前市面上最主流、最好用的几个替代/升级版开源项目：

## 1. 综合体验最好、最推荐：3K-Vless-Pages

如果说 `edgetunnel`​ 是初代神器，那 `3K-Vless-Pages`​（或者社区里各种衍生优化版，如 `fyl080801/Cloudflare-workers-pages-vless`）就是目前的集大成者。

- **核心优势：**

  - **自带全平台订阅转换：**  `edgetunnel`​ 最麻烦的是配置好后要手动拼节点链接。而这个项目部署好后，访问网页端就能直接生成 **Clash、Sing-box、v2rayN、Shadowrocket（小火箭）**  的直接订阅链接。
  - **完美支持 Pages 部署：**  除了 Workers，它原生支持部署在 CF Pages 上。Pages 的额度相对更宽松，且自带免费自定义域名（不容易被墙）。
  - **内置优选 IP 自动分发：**  可以在后台直接配置多组 CF 官方 IP 或反代 IP，客户端更新订阅时会自动同步最新的优选 IP。

## 2. 功能最硬核、适合进阶：BPB-Panel (Bypass Censorship)

这是一个由海外大牛开发的、基于 Cloudflare Workers 的超级面板项目。

- **核心优势：**

  - **极其精美的可视化后台：**  部署完成后，它提供了一个完整的 Web 登录面板（类似路由器后台）。
  - **多协议支持：**  除了标准的 VLESS，它还融合了多种抗封锁和混淆策略。
  - **内置多款优选 IP 测速结果：**  面板内集成并实时更新全球不同地区的 CF 优选节点，甚至能根据你的地区自动匹配延迟最低的 IP。
  - **一键生成订阅：**  同样支持一键导入各大主流客户端。

## 3. 最轻量、最接近 edgetunnel 的纯净版：cmliu 系列 / vless-cloudflare

如果你不喜欢花里胡哨的网页面板，只想要一段干净、高效、没有后门的纯代码，推荐关注 GitHub 上 **cmliu** 或者是 ​**zizifn**（edgetunnel 原作者之后）的修改版。

- **核心优势：**

  - **代码极其精简：**  只有几百行代码，没有任何第三方外链，不收集任何数据，执行效率极高（CPU 占用低）。
  - **修复了原版的各种 Bug：**  完美解决了原版 `edgetunnel` 容易断流、对部分特殊 TLS 握手支持不佳的问题。
  - **支持 ProxyIP：**  原生支持配置 `ProxyIP`（即用 CF 节点去中转那些被 CF 自身屏蔽的网站，如 ChatGPT）。

## 💡 终极建议

如果你觉得 `edgetunnel`​ 用着有点累（比如要自己拼节点、容易断流），推荐立刻去 GitHub 搜索并部署 ​**​`Cloudflare-workers-pages-vless`​**​ **（3K或衍生版）** 。

**部署小贴士：**  尽量选择 **Pages 部署** 方式（把代码放到 GitHub 私人仓库，然后用 CF Pages 关联绑定）。这样不仅可以使用 CF 赠送的 `.pages.dev`​ 免费子域名（这个域名目前在国内的被墙概率远低于 Workers 的 `.workers.dev`），而且更新代码只需要在 GitHub 上一键同步即可，非常优雅。
