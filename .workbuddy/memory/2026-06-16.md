# 2026-06-16 工作日志

## B/C 模块滚动掉帧优化
- **文件**: `src/pages/index.astro`
- **根因**：B 模块（Axis Service Cards 4张）和 C 模块（探索领域 4张）共 8 张卡片使用了 `backdrop-filter: blur(20px)`，每次滚动浏览器需对 8 个独立合成层做模糊重绘，GPU 压力极高。
- **修复**：
  1. 移除所有 8 张卡片的 `backdrop-filter: blur(20px)` + `-webkit-backdrop-filter`，替换为 `background: rgba(20, 20, 20, 0.7)` 纯色半透明背景
  2. `transition: all 0.4s` → 精确属性列表 `transform, border-color, background, box-shadow`，避免触发不必要的重绘
  3. 移除 `.axis-services-portrait` 的 `filter: grayscale(1) brightness(0.7) contrast(1.3)`，改 `background-size: cover` 替代 `contain`
  4. 隐藏 `.axis-services-glitch`（`mix-blend-mode: overlay` + `repeating-linear-gradient` 叠加层）
